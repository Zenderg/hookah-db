# Security Policy

## Supported Version

Пока проект не имеет публичных релизных веток. Security fixes принимаются в `main`.

## Reporting a Vulnerability

Если вы нашли уязвимость, не публикуйте exploit в issue. Создайте приватное сообщение автору репозитория или откройте минимальный issue без технических деталей, чтобы согласовать безопасный канал связи.

## Self-hosting Notes

- Не используйте `local-dev-key` в production. Он создаётся только sample seed командой и предназначен для локального запуска.
- Храните `.env` вне git и задавайте production secrets через вашу систему деплоя.
- `PARSER_CRON_ENABLED=false` рекомендуется для новых окружений, пока вы явно не решили включить daily импорт.
- Парсер работает со сторонним сайтом. Используйте лимиты, backoff и ручной запуск для диагностики, чтобы не создавать лишнюю нагрузку.
